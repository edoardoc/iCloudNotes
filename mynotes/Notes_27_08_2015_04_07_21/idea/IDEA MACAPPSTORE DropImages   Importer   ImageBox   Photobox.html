FEATURES:
1. import immagini da fotocamera e salva nel folder predefinito
deve essere estremamente semplificato, partire dai default e rendere i default modificabili

2. sincronizzare le foto con una lista di repository SOLO FOLDERS che possono far capo a cartelle condivise, NAS, dischi USB, ecc.
a. tutto deve funzionare senza intervento: se un device è scollegato si deve dare solo notifica alla fine ma comunque procedere!
b. se manca spazio in un device si interrompe la copia SU QUEL DEVICE ANNULLANDO TUTTO IL FOLDER
l'unico alert in rosso che va dato è se l'utente trasferisce le foto SPOSTANDOLE su UN SOLO REPO che magari è SCOLLEGATO
DA RIVEDERE c. il sync va in default assumendo che la CAMERA E' UN REPOSITORY - tipo iPhone - (a meno che l'utente specifica di SPOSTARE le foto) 
d. l'operazione atomica non è il file ma bensi' il folder - DA DECIDERE BENE (se qualcosa va male durante il sync si annulla tutto quel ramo di sync senza lasciare file sparsi)
e. ci sono due tipi di repository: statico (file zip) e normale (folder)


3. la fase di sync avviene a LIVELLO DI FILE (magari rsync?) e se ne frega se si tratta di immagini o altro, fa i confronti con il nome / date / size (eventualmente anche CRC da setting)
4. (advanced) alcuni REPO possono essere LOW-RES e contenere delle versioni a bassa risoluzione delle foto per potersele portare dietro su device con poca memoria
